C# におけるクロージャの挙動と注意点（面接向け説明）

C# のクロージャは、値のコピーではなく “変数そのもの（参照）” を捕捉する という特徴があります。
そのため、以下のような問題が発生する可能性があります。

■ クロージャは参照を保持する

クロージャ内部で使用されているローカル変数は、
そのスコープを抜けた後でも ガベージコレクションの対象にならず保持され続けます。

■ クラス内でクロージャが参照を捕捉している場合

もしクラス内のメンバー変数をクロージャが捕捉している場合、
そのクラスインスタンスを明示的に破棄（解放）しない限り、

クロージャ → クラスインスタンス

クラスインスタンス → 捕捉された変数

という参照関係が残り続け、
本来解放されるべきオブジェクトが長期間生存してしまう（メモリリークのような状態） になります。

■ 対策

そのため、不要になったタイミングで

クラスインスタンスを 明示的に null に設定する

クロージャを保持しているデリゲートやイベントを 解除する

など、参照を断ち切る必要があります。



GC后引用数据的地址会不会改变？如何保持不变？
在C#中，无论是强引用类型还是其他引用类型，在垃圾收集（GC）过程中都可能发生内存地址的变化。GC的主要任务之一是整理内存空间，减少内存碎片，这通常通过移动对象来实现。因此，即使是强引用类型，其内存地址也可能会在GC后变化。
使用fixed关键字固定住。

C# における GC とオブジェクトのアドレス変動について（面接向け説明）

C# のマネージド環境では、GC（Garbage Collector）がオブジェクトを移動する可能性があるため、
通常は参照型オブジェクトのメモリアドレスが GC 後に変わることがあります。

■ GC によるオブジェクト移動

.NET の GC はメモリ断片化を防ぐため、
コンパクション（メモリの整理）を行う際に ヒープ上のオブジェクトを別の場所へ移動 します。
そのため、強参照を持っている場合でも アドレスそのものは固定されている訳ではありません。

🌙 アドレスを固定したい場合の方法
■ fixed キーワードを使用する

C# では fixed キーワードを使用することで、
GC によるオブジェクト移動を一時的に禁止し、オブジェクトのアドレスを固定できます。

fixed (int* ptr = &someStruct.value)
{
    // このスコープ内ではアドレスが固定される
}


fixed ブロック内ではオブジェクトが “ピン止め（pinning）” される

ピン止め中は GC による移動が禁止され、アドレスが変わらないことが保証される

🌙 補足：Pinning の注意点

ピン止めされたオブジェクトは GC の最適化を妨げることがある

長時間の pinning はヒープの断片化につながるため、
必要最小限のスコープで使用することが推奨されます
